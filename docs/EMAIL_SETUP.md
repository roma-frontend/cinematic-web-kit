# Почта проекта: бесплатно, без лимитов на приём, с info@ / support@ / sales@

Эта инструкция настраивает **всю** почтовую инфраструктуру Cinematic Web Kit
на бесплатных тарифах:

| Задача | Сервис | Цена | Лимиты |
|---|---|---|---|
| Приём писем на `info@`, `support@`, `sales@` | Cloudflare Email Routing | бесплатно навсегда | **без лимитов** |
| Отправка транзакционных писем (код входа, сброс пароля) | Resend | бесплатно | 3 000 писем/мес |
| Резервный отправитель (failover — включается сам) | Brevo | бесплатно | 300 писем/день (~9 000/мес) |

Транзакционных писем у проекта два вида — 6-значный код при входе и ссылка
сброса пароля. Для них бесплатных квот Resend + Brevo (≈12 000/мес суммарно)
хватает с большим запасом, а приём почты через Cloudflare действительно
безлимитный.

---

## Шаг 1. Приём: info@, support@, sales@ через Cloudflare Email Routing

Подходит, если домен обслуживается в Cloudflare (у проекта уже есть аккаунт —
используется Web Analytics beacon).

1. Cloudflare Dashboard → ваш домен → **Email** → **Email Routing** → *Get started*.
2. Cloudflare сам добавит нужные MX и SPF записи — подтвердите одним кликом.
3. **Destination addresses**: добавьте свой реальный ящик (например, Gmail)
   и подтвердите его по письму.
4. **Custom addresses** — создайте три адреса:
   - `info@ваш-домен` → ваш ящик
   - `support@ваш-домен` → ваш ящик
   - `sales@ваш-домен` → ваш ящик

   (можно направить их в разные ящики или включить Catch-all.)

Готово: входящая почта на все три адреса пересылается бесплатно и без лимитов.

**Отвечать от имени этих адресов** можно прямо из Gmail:
Настройки → Аккаунты и импорт → «Отправлять письма как» → добавьте
`support@ваш-домен` (SMTP: `smtp.gmail.com`, либо SMTP Brevo из шага 3).

> Альтернатива с полноценными ящиками (не пересылкой): **Zoho Mail Forever Free**
> — до 5 пользователей на своём домене бесплатно (веб-интерфейс + IMAP/POP).
> Подойдёт, если нужны отдельные входящие для отдела продаж и поддержки.

## Шаг 2. Отправка: Resend (основной провайдер)

1. Зарегистрируйтесь на [resend.com](https://resend.com) (бесплатно, карта не нужна).
2. **Domains** → *Add Domain* → ваш домен → добавьте предложенные DNS-записи
   (DKIM/SPF; в Cloudflare это две-три записи TXT/CNAME) → дождитесь Verified.
3. **API Keys** → *Create API Key* → скопируйте ключ.
4. В `.env.local`:

   ```ini
   RESEND_API_KEY=re_ваш_ключ
   EMAIL_FROM=info@ваш-домен
   EMAIL_FROM_NAME=Cinematic Web Kit
   EMAIL_REPLY_TO=support@ваш-домен
   ```

Всё. С этого момента:
- при входе пользователю приходит фирменное письмо с 6-значным кодом;
- «Забыли пароль?» отправляет письмо со ссылкой сброса (действует 60 минут).

## Шаг 3 (опционально). Brevo — резерв и запас квоты

1. Зарегистрируйтесь на [brevo.com](https://www.brevo.com) (бесплатный тариф).
2. **Senders & Domains** → подтвердите домен (DKIM-записи в Cloudflare)
   или хотя бы отправителя `info@ваш-домен`.
3. **SMTP & API** → *API Keys* → создайте ключ и добавьте:

   ```ini
   BREVO_API_KEY=xkeysib-ваш_ключ
   ```

Логика в `lib/email.ts`: сначала пробуется Resend, при ошибке — Brevo
(порядок меняется переменной `EMAIL_PROVIDER=brevo`). Если письмо не удалось
отправить вообще, вход деградирует до обычного парольного — пользователи
никогда не окажутся заблокированы из-за сбоя почты.

## Рекомендованные DNS-записи (антиспам)

Resend/Brevo дадут свои DKIM/SPF записи автоматически. Дополнительно добавьте
DMARC (одна TXT-запись, повышает доставляемость):

```
_dmarc.ваш-домен  TXT  "v=DMARC1; p=none; rua=mailto:info@ваш-домен"
```

## Поведение без настройки

| Ситуация | Что происходит |
|---|---|
| Ключей нет (dev/тесты) | Письма печатаются в консоль сервера; код при входе **выключен** — вход по одному паролю, как раньше |
| Ключ есть, но провайдер упал | Failover на второго провайдера; если упали оба — вход без кода (событие пишется в аудит `auth.otp_send_failed`) |
| Нужно отключить код при входе | `AUTH_EMAIL_OTP=off` |

## Что письма из себя представляют

Шаблоны в `lib/email-templates.ts` свёрстаны в фирменном стиле проекта:
тёмный «кинематографичный» фон, индиго-градиент `#3c68d9 → #22b0d0` (те же
oklch-токены, что в `app/globals.css`), логотип-бейдж ▶, контакты
`info@ / support@ / sales@` в подвале (подставляются из домена `EMAIL_FROM`).
Вёрстка email-safe: таблицы, инлайн-стили, без внешних картинок — корректно
рендерится в Gmail, Outlook и Apple Mail, включая тёмную тему.
